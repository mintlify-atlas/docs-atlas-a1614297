## Overview

tmux uses a comprehensive style system for customizing the appearance of status lines, window lists, panes, and other UI elements. Styles support colors, text attributes, alignment, and more.

## Style Syntax

Styles are specified as comma-separated attribute pairs in the format `attribute=value`:

```bash
set -g status-style bg=blue,fg=white,bold
set -g window-status-style bg=default,fg=yellow
```

### Foreground and Background

```bash
fg=red           # Foreground (text) color
bg=blue          # Background color
us=green         # Underscore color (if supported)
```

<Note>
The value `8` or `default` resets a color to the terminal default.
</Note>

## Text Attributes

### Available Attributes

Apply text styling with these attributes:

<CodeGroup>
```bash Font Weight and Style
bold             # Bold text
dim              # Dimmed text
italics          # Italic text (if supported)
```

```bash Text Decoration
underscore       # Underlined text
overline         # Overlined text (if supported)

```

```bash Visual Effects
blink            # Blinking text
reverse          # Reverse video (swap fg/bg)

```
</CodeGroup>

### Setting Attributes

<CodeGroup>
```bash Enable Attributes
set -g status-style fg=white,bg=black,bold
set -g message-style fg=red,bold,reverse
```

```bash Disable Attributes
set -g status-style fg=white,nobold,noitalics
set -g pane-border-style fg=blue,noreverse
```

```bash Clear All Attributes
set -g status-style fg=white,none
# 'none' clears all attributes
```

```bash Special Attribute
set -g status-style noattr
# noattr prevents any attributes from being applied
```
</CodeGroup>

## Fill Color

Set a fill color for padding in status line elements:

```bash
set -g status-left-style fill=blue
set -g status-right-style fill=default
```

The fill color is used for any padding added to status elements.

## List Styles

Special styles for list items in choose mode:

<CodeGroup>

list=focus       # Style when item is focused

## Range Styles

Apply styles to specific ranges in the status line for mouse interactions:

<CodeGroup>
```bash Range Types
range=left       # Left section of status line
range=right      # Right section of status line
range=window|N   # Window with index N
range=pane|%N    # Pane with ID %N
range=session|$N # Session with ID $N
range=user|name  # Custom named range
```

```bash Examples
set -g status-left-style range=left
set -g window-status-style range=window|0
```
</CodeGroup>

<Warning>
Range styles are primarily used internally for mouse event handling. Most users won't need to set these directly.
</Warning>

## Inline Style Changes

Use `#[style]` to change styles inline in format strings:

<CodeGroup>
```bash Color Changes
set -g status-left "#[fg=green]Session: #[fg=yellow]#{session_name}"
```

```bash Attribute Changes
set -g status-right "#[bold]#(date)#[nobold] | #H"
```

```bash Combined Changes
set -g window-status-format "#[fg=white,bg=black] #I:#W "
set -g window-status-current-format "#[fg=black,bg=green,bold] #I:#W "
```

```bash Reset to Default
set -g status-left "Prefix #[default]Normal"
# #[default] resets all styles
```
</CodeGroup>

## Parsing Style Strings

When tmux parses a style string:

1. It splits on commas and whitespace
2. Each token is checked against known attributes
3. `fg=`, `bg=`, and `us=` set colors
4. Named attributes enable features
5. `no` prefix disables attributes
6. `none` clears all attributes
7. `default` resets to base style

## Converting Colors to RGB

Use the `#{c:color}` format to convert tmux colors to hexadecimal RGB:

```bash
display-message -p "#{c:red}"        # Outputs: #ff0000
display-message -p "#{c:colour123}"  # Outputs hex RGB value
```

<Note>
Not all terminals support all style features. Features like `overline`, `strikethrough`, and RGB colors depend on terminal capabilities. Use the `terminal-features` option to configure supported features.
</Note>

## Style Inheritance

Styles can be built incrementally using the `-a` flag:

```bash
set -g status-style bg=red
set -ag status-style fg=blue    # Adds fg=blue, keeps bg=red
```

Without `-a`, the new style replaces the old one:

```bash
set -g status-style bg=red
set -g status-style fg=blue     # Only fg=blue, bg is reset to default
```
