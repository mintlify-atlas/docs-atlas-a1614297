## Overview

The status line displays information about your tmux session at the top or bottom of the screen. It's highly customizable using format strings, styles, and dynamic updates.

## Status Line Architecture

From `status.c:227-264`, the status line system:

<Tabs>
  <Tab title="Structure">
    The status line consists of multiple components:
    
    - **Position** - Top or bottom of screen
    - **Lines** - Single or multiple status lines
    - **Left section** - Session information (default)
    - **Center/Windows** - Window list
    - **Right section** - Time/date (default)
    - **Style** - Colors and attributes
  </Tab>

  <Tab title="Update Mechanism">
    ```c
    void status_timer_start(struct client *c)
    {
        struct session *s = c->session;

        if (event_initialized(&c->status.timer))
            evtimer_del(&c->status.timer);
        else
            evtimer_set(&c->status.timer, status_timer_callback, c);

        if (s != NULL && options_get_number(s->options, "status"))
            status_timer_callback(-1, 0, c);
    }
    ```
    
    The status line updates automatically based on the `status-interval` option.
  </Tab>
</Tabs>

## Basic Configuration

### Enable/Disable Status Line

```bash
# Enable status line (default: on)
set -g status on

# Disable status line
set -g status off

# Number of status lines (1-5)
set -g status 2
```

### Position

```bash
# Position at bottom (default)
set -g status-position bottom

# Position at top
set -g status-position top
```

### Update Interval

From `status.c:195`, the interval controls refresh rate:

```bash
# Update every second
set -g status-interval 1

# Update every 5 seconds (default)
set -g status-interval 5

# Disable automatic updates (0)
set -g status-interval 0
```

## Styling

### Colors

<Accordion title="Color Options">

**Named colors:**
- black, red, green, yellow, blue, magenta, cyan, white
- brightred, brightgreen, brightyellow, etc.
- default (terminal default)

**Numbered colors:**
- colour0 to colour255 (256-color palette)
- Use `#RRGGBB` for true color (requires terminal support)

**Example:**
```bash
set -g status-style "fg=colour136,bg=colour235"
```

</Accordion>

### Attributes

<CodeGroup>

```bash Text Attributes
# Single attribute
set -g status-style "bold"

# Multiple attributes (comma-separated)
set -g status-style "bold,underscore"

# Available attributes:
# - bold
# - dim
# - underscore (or underline)
# - blink
# - reverse
# - italics
# - strikethrough
```

```bash Combined Style
# Foreground, background, and attributes
set -g status-style "fg=white,bg=black,bold"

# With default values
set -g status-style "fg=default,bg=colour235,none"
```

</CodeGroup>

## Format Strings

### Common Variables

<Tabs>
  <Tab title="Session">
    ```bash
    #S    # Session name
    #D    # Session ID
    #{session_attached}    # Number of attached clients
    #{session_created}     # Session creation time
    #{session_windows}     # Number of windows
    ```
  </Tab>

  <Tab title="Window">
    ```bash
    #I    # Window index
    #W    # Window name
    #T    # Window title
    #F    # Window flags (*, -, Z, etc.)
    #{window_id}           # Window ID (@1, @2, etc.)
    #{window_panes}        # Number of panes
    #{window_active}       # 1 if window is active
    ```
  </Tab>

  <Tab title="Pane">
    ```bash
    #P    # Pane index
    #{pane_id}             # Pane ID (%1, %2, etc.)
    #{pane_current_path}   # Current directory
    #{pane_current_command}# Running command
    #{pane_title}          # Pane title
    #{pane_width}          # Pane width
    #{pane_height}         # Pane height
    ```
  </Tab>

  <Tab title="Host/Time">
    ```bash
    #H    # Hostname
    #h    # Hostname (short)
    %Y    # Year (4 digits)
    %m    # Month (01-12)
    %d    # Day (01-31)
    %H    # Hour (00-23)
    %M    # Minute (00-59)
    %S    # Second (00-59)
    %a    # Weekday (short)
    %A    # Weekday (full)
    ```
  </Tab>
</Tabs>

## Practical Examples

### Minimal Status Line

```bash
# Simple and clean
set -g status-style "bg=default,fg=white"
set -g status-left "#S "
set -g status-right "%H:%M"
set -g window-status-format " #I:#W "
set -g window-status-current-format " #I:#W "
set -g window-status-current-style "bold"
```

### Powerline-style Status

```bash
# Powerline-inspired status line
set -g status-style "bg=colour234,fg=colour137"
set -g status-left "#[fg=colour232,bg=colour154,bold] #S #[fg=colour154,bg=colour238,nobold]#[fg=colour222,bg=colour238] #(whoami) #[fg=colour238,bg=colour234,nobold]"
set -g status-right "#[fg=colour238,bg=colour234]#[fg=colour222,bg=colour238] %H:%M #[fg=colour154,bg=colour238]#[fg=colour232,bg=colour154,bold] #H "
set -g window-status-format "#[fg=colour234,bg=colour238]#[fg=white,bg=colour238] #I #[fg=white,bg=colour238]#W #[fg=colour238,bg=colour234]"
set -g window-status-current-format "#[fg=colour234,bg=colour39]#[fg=colour232,bg=colour39,noreverse,bold] #I #[fg=colour232,bg=colour39,bold]#W #[fg=colour39,bg=colour234,nobold]"
```

### Information-Rich Status

```bash
# Show comprehensive information
set -g status-left-length 40
set -g status-right-length 80

set -g status-left "#[fg=green]#S #[fg=yellow]#I:#P "
set -g status-right "#[fg=cyan]#(tmux-mem-cpu-load --interval 2) #[fg=yellow]%Y-%m-%d #[fg=green]%H:%M:%S"

set -g window-status-format "#I:#W#{?window_flags,#{window_flags}, }"
set -g window-status-current-format "#[fg=black,bg=green]#I:#W#{?window_flags,#{window_flags}, }"
```

### Conditional Nested Status

From `example_tmux.conf:11-14`:

```bash

# Different colors per host
```

## Dynamic Updates

### Running Shell Commands

```bash
# Run command and display output
set -g status-right "#(uptime | cut -d',' -f 3-)"

# With refresh interval
set -g status-interval 2
set -g status-right "Load: #(cat /proc/loadavg | cut -d' ' -f1)"

# Multiple commands
set -g status-right "CPU: #(top -bn1 | grep 'Cpu' | cut -d',' -f1 | awk '{print $2}')% | Mem: #(free -h | awk '/^Mem:/ {print $3"/"$2}')"
```

<Tip>
Shell commands in status-right/status-left run every status-interval seconds. Keep them lightweight to avoid performance issues.
</Tip>

### Custom Scripts

```bash
# Use custom script for status
set -g status-right "#(~/.tmux/scripts/status.sh)"

# Script example: ~/.tmux/scripts/status.sh
#!/bin/bash
load=$(uptime | awk '{print $(NF-2)}' | sed 's/,//')
mem=$(free -h | awk '/^Mem:/ {print $3"/"$2}')
echo "Load: $load | Mem: $mem | %H:%M"
```

## Troubleshooting

### Status Line Not Updating

<Steps>

<Step title="Check Interval">
```bash
# Verify status-interval is not 0
tmux show-options -g status-interval

# Set interval if needed
tmux set -g status-interval 5
```
</Step>

<Step title="Force Refresh">
```bash
# Manually refresh
tmux refresh-client -S
```
</Step>

<Step title="Check Status Enabled">
```bash
# Verify status is on
tmux show-options -g status
```
</Step>

</Steps>

### Format String Errors

```bash
# Test format string
tmux display-message -p "#{session_name}"

# Show all format variables
tmux list-keys -T copy-mode | head -1
tmux display-message -p "#{?window_active,active,inactive}"
```

<Accordion title="Shell Command Debugging">

```bash
# Test command independently
$ uptime | cut -d',' -f 3-

# Check shell in tmux
tmux show-options -g default-shell

# Verify command output
tmux set -g status-right "#(echo test)"

# Check command execution
# Commands run with: /bin/sh -c "command"
```

</Accordion>

## Performance Tips

<Tip>
**Optimize Status Line Performance:**

1. Increase `status-interval` to reduce updates
2. Minimize shell command complexity
3. Cache expensive operations
4. Use format strings over shell commands when possible
5. Limit status-left-length and status-right-length

```bash
# Good: Native format
set -g status-right "#{session_windows} windows | %H:%M"

# Less efficient: Shell command every interval
set -g status-right "#(tmux list-windows | wc -l) windows | %H:%M"
```
</Tip>
